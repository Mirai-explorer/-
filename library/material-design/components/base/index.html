基础

MDC Base 包含核心基础类和组件类，它们分别充当所有 MDC Web 的基础类和组件的基类。

大多数时候，您不必直接依赖于 mdc-base。但是，如果您希望编写遵循 MDC Web 模式并与 MDC Web 生态系统优雅地集成的自定义组件，那么这将非常有用。

安装

首先要安装的模块：

 
npm install @material/base

然后通过以下方式之一将其嵌入到您的代码中：

ES Module 语法

 
import {MDCComponent, MDCFoundation} from '@material/base';

CommonJS

 
const MDCComponent = require('mdc-base').MDCComponent;
const MDCFoundation = require('mdc-base').MDCFoundation;

AMD

 
require(['path/to/mdc-base'], function(mdcBase) {
  const MDCComponent = mdcBase.MDCComponent;
  const MDCFoundation = mdcBase.MDCFoundation;
});

Vanilla

 
const MDCComponent = mdc.base.MDCComponent;
const MDCFoundation = mdc.base.MDCFoundation;

用法

mdc-base 公开了两个类： MDCComponent （默认导出），所有组件都从它扩展而来； MDCFoundation，所有基础类都从它扩展而来。欲知更多关于基础类和组件的信息，请查看我们对体系结构和最佳实践的概述。

MDCFoundation

MDCFoundation 提供了实现基础类的基本机制。子类需要：

  • 在需要时提供适当的静态 gatters 实现。
  • 提供 init() 和 destroy() 生命周期方法
​x
 
import {MDCFoundation} from '@material/base/foundation';
​
export default class MyFoundation extends MDCFoundation {
  static get cssClasses() {
    return {
      ROOT: 'my-component',
      MESSAGE: 'my-component__message',
      BUTTON: 'my-component__button',
      TOGGLED: 'my-component--toggled'
    };
  }
​
  static get defaultAdapter() {
    return {
      toggleClass: (/* className: string */) => {},
      registerBtnClickHandler: (/* handler: Function */) => {},
      deregisterBtnClickHandler: (/* handler: Function */) => {}
    };
  }
​
  constructor(adapter) {
    super({...MyFoundation.defaultAdapter, ...adapter});
    const {TOGGLED} = MyFoundation.cssClasses;
    this.clickHandler_ = () => this.adapter_.toggleClass(TOGGLED);
  }
​
  init() {
    this.adapter_.registerBtnClickHandler(this.clickHandler_);
  }
​
  destroy() {
    this.adapter_.deregisterBtnClickHandler(this.clickHandler_);
  }
}

静态 Getters

静态 getters 指定了可在基础类及其组件还有第三方代码中使用的常量。 重要的是要记住总是将常量放入这些 getters 中。这将确保您的组件可以在尽可能多的环境中进行互操作，包括那些需要由宿主库覆盖 CSS 类的环境（例如 Closure Stylesheets），或是需要修改字符串的环境（比如用于 i18）。

注意，如果您的组件没有 getters，则不必显式地为常量提供 getters。

需要提供的 getters 方法如下：

getter 说明
cssClasses 返回一个对象，其中每个键标识一些代码将依赖的 CSS 类。
strings 返回一个对象，其中每个键标识一个字符串常量，例如 ARIA_ROLE
numbers 返回一个对象，其中每个键标识一个数字常量，例如 TRANSITION_DELAY_MS
defaultAdapter 返回指定适配器形状的对象。可以作为适配器的合理默认值使用，也可以指定适配器的“schema”

生命周期方法

每个基础类都有两个生命周期方法： init() and destroy()，如下所述：

方法 调用时间 用例
init() 组件预初始化时由宿主类调用 添加事件监听器，通过适配器查询信息等。
destroy() 组件不再使用时由宿主类调用 移除事件侦听器，重置一切瞬态等。

MDCComponent

MDCComponent 提供了实现组件类的基本机制。

 
import MyComponentFoundation from './foundation';
​
export class MyComponent extends MDCComponent {
  static attachTo(root) {
    return new MyComponent(root);
  }
​
  getDefaultFoundation() {
    const btn = this.root.querySelector(`.${MyComponentFoundation.cssClasses.BUTTON}`);
    return new MyComponentFoundation({
      toggleClass: className => {
        if (this.root.classList.contains(className)) {
          this.root.classList.remove(className);
          return;
        }
        this.root.classList.add(className);
      },
      registerBtnClickHandler: handler => btn.addEventListener('click', handler),
      deregisterBtnClickHandler: handler => btn.removeEventListener('click', handler)
    });
  }
}

Properties

MDCComponent 为子类提供以下“私有”属性：

属性 说明
root_ 根元素作为第一个参数传入到构造函数中。
foundation_ 此组件的基础类。它可作为可选的第二个参数传递给构造函数，也可将调用 getDefaultFoundation() 的结果赋值给它。

方法

MDCComponent 为子类提供以下方法：

方法 描述
initialize(...args) 在根元素附加到组件之后，基础实例化之前调用。在根元素之后传递给组件构造函数的所有位置参数，以及可选的 foundation 第二参数，都将会提供此方法。这是在构造函数中进行任意设置工作的好地方。
getDefaultFoundation() 返回为组件正确配置的基础类的实例。在构造函数内未提供基础实例时调用。子类必须实现此方法。
initialSyncWithDOM() 在构造函数中调用。如果子类希望与宿主 DOM 元素执行初始状态同步，则可以重写此方法。例如，滑块可能想要检查其宿主元素是否包含一个预设值，并相应地调整其内部状态。注意，这一方法注意事项与基类生命周期方法相同。默认为 no-op（无操作）。
destroy() 如果子类希望在组件被销毁时执行任何额外的清理工作，则可以重写此方法。例如，组件可能希望取消对窗口大小调整侦听器的注册。
listen(type: string, handler: EventListener) 将事件侦听器添加到给定类型的组件的根节点。注意，这只是 this.root_.addEventListener 的一个根节点。
unlisten(type: string, handler: EventListener) 从组件的根节点移除事件侦听器。注意，这只是 this.root_.removeEventListener 的一个代理。
emit(type: string, data: Object, shouldBubble: boolean = false) 从组件根节点分派一个包含具体 data ，类型为 type 的自定义事件。它还使用一个可选的 shouldBubble 参数来指定事件是否应该冒泡。这是在我们原始组件中调度事件的首选方法。

静态方法

除继承方法外，子类还应在其代码内实现以下两个静态方法：

方法 说明
attachTo(root) => <ComponentClass> 子类必须实现此方法，以便使用提供的根元素实例化和返回类的实例。其将在 mdc-auto-init 中使用，将来可以通过自定义 lint 规则来强制使用。

基础生命周期处理

MDCComponent 在其构造函数中调用 init() 函数，在其destroy()函数中调用 destroy() 函数。因此，在重写 destroy() 时始终调用 super()，这点非常重要。否则可能会导致资源泄漏。

初始化和构造函数参数

如果需要向组件的构造函数传递额外的参数，可以使用 initialize 方法，如上所示。这方面的一个例子 是将子组件作为依赖项传入。

 
class MyComponent extends MDCComponent {
  initialize(childComponent = null) {
    this.child_ = childComponent ?
      childComponent : new ChildComponent(this.root_.querySelector('.child'));
  }
​
  getDefaultFoundation() {
    return new MyComponentFoundation({
      doSomethingWithChildComponent: () => this.child_.doSomething(),
      // ...
    });
  }
}

您可以这样调用此代码：

 
const childComponent = new ChildComponent(document.querySelector('.some-child'));
const myComponent = new MyComponent(
  document.querySelector('.my-component'), /* foundation */ undefined, childComponent
);
// use myComponent

注意：如果您愿意的话，您也可以传递一个初始化的基础。上面的示例简单地展示了如何在不实例化 foundation 的情况下传递初始化参数。

最佳实践：简化适配器

如果您发现您的适配器变得过于复杂，则应考虑将复杂的部分重构为它们自己的 implementations（实现）。

 
import MyComponentFoundation from './foundation';
import {toggleClass} from './util';
​
class MyComponent {
  // ...
  getDefaultFoundation() {
    return new MyComponentFoundation({
      toggleClass: className => util.toggleClass(this.root_, className),
      // ...
    });
  }
}

其中 ./util 可以是：

 
export function toggleClass(element, className) {
  if (root.classList.contains(className)) {
    root.classList.remove(className);
    return;
  }
  root.classList.add(className);
}

这样不但降低了组件类的复杂性，还允许对复杂适配器的功能进行充分的测试：

 
test('toggleClass() removes a class when present on an element', t => {
  const root = document.createElement('div');
  root.classList.add('foo');
​
  util.toggleClass(root, 'foo');
​
  t.false(root.classList.contains('foo'));
  t.end();
});
